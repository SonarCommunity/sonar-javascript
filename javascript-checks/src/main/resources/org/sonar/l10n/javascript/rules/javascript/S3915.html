Conditional statements using an expression which cannot evaluate to anything but false have the effect of making blocks of code non-functional. It is quite likely that the code does not match the programmer's intent.

Noncompliant Code Example

if (a && b) {
  if (!a) { // Noncompliant, always false
    // ...
  }
}

See

  • MISRA C:2004, 13.7 - Boolean operations whose results are invariant shall not be permitted.
  • MISRA C:2012, 14.3 - Controlling expressions shall not be invariant
  • MITRE, CWE-489 - Leftover Debug Code
  • MITRE, CWE-570 - Expression is Always False
  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed
  • CERT, MSC12-CPP. - Detect and remove code that has no effect
