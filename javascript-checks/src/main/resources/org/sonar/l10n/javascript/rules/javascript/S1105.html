Shared naming conventions allow teams to collaborate effectively. This rule raises an issue when:

* an open curly brace is not placed at the end of a line of code.

* the closing brace is not on the same line as the else, if-else, catch or finally keyword.

* a statements inside brace appears at the same line as the opening or closing brace.

Noncompliant Code Example

if (condition)
{                                                      //Noncompliant
  doSomething();
}                                                      //Noncompliant
else {
  doSomethingElse();
}

try { somethingRisky();                            //Noncompliant
} catch (err) {
  handleError();
}{code}

Compliant Solution

if (condition) {                                   //Compliant
  doSomething();
} else {                                           //Compliant
  doSomethingElse();
}

try {
  somethingRisky();                                //Compliant
} catch (err) {
  handleError();
}

Exceptions

* Object literals appearing as arguments can start on their own line.

functionWithObject(
   {                                                 //Compliant
        g: "someValue"
   }
);

* When blocks are inlined (left and right curly braces on the same line), no issue is triggered.

if(condition) {doSomething();}                       //Compliant
