Using a comparison operator >, <, >=, <= when at least one operand is an Object or undefined will always result in false value.

Noncompliant Code Example

x = undefined;
if (y > x) {}  //Non-compliant [BUG] > condition is false whatever is the "y" value

Exceptions

- Comparisons of two Date objects are ignored.

- The cases obj <= obj and obj >= obj, where obj is an Object, are ignored as they always return true.
