Using a comparison operator >, <, >=, <= will always result in false value in the cases below.

1. At least one of the two operands is undefined.

2. One operand is convertible to a number and the other operand is an Object which is not convertible to a number.

Note that this implies that that a Date can be compared with a Boolean or a Number or another Date or null.

Noncompliant Code Example

x = undefined;
if (y > x) {}  //Non-compliant [BUG] > condition is false whatever is the "y" value

obj = new Object();
if (obj > 34) {}  //Non-compliant
