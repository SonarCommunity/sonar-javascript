There is no good excuse for an empty class. If it’s being used simply as a common extension point, it should be replaced with an interface. If it was stubbed in as a placeholder for future development it should be fleshed-out. In any other case, it should be eliminated.

Additionnaly, one shouldn’t use a class to store exclusively static methods. In JavaScript, you can use a module, or better export each method separately.

Why is this an issue?

Using an empty class serves no purpose and can hinder the readability of the code.

class Foo {
  static bar: () => {
    // ...
  }
}

How to fix it

You can simply directly import the functions that you wish to make available.

export bar = () => {
  // ...
}

Code examples

Noncompliant code example

class Foo {
  static bar: () => {
    // ...
  }
}

class DoAndLog {
  constructor () {
    console.log('I\'m done!');
  }
}

Compliant solution

export bar = () => {
  // ...
}

function doAndLog() {
  console.log('I\'m done!');
}
