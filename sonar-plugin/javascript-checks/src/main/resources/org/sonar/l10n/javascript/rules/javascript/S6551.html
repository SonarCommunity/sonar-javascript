When calling toString() or coercing into a string an object that doesnâ€™t implement its own toString method, it returns [object Object] which is often not what was intended.

Why is this an issue?

Printing such an object will not express the information contained in the object, providing unexpected results.

How to fix it

You can simply define a toString() method for the object or class.

Code examples

Noncompliant code example

class Foo {};
const bar = new Foo();

bar + '';
`Foo: ${bar}`;
bar.toString();

Compliant solution

class Foo {
  toString() {
    return 'Foo';
  }
}
const bar = new Foo();

bar + '';
`Foo: ${bar}`;
bar.toString();

Noncompliant code example

const foo = {};
foo + '';
`Foo: ${foo}`;
foo.toString();

Compliant solution

const foo = {
  toString: () => {
    return 'Foo';
  }
}
foo + '';
`Foo: ${foo}`;
foo.toString();
