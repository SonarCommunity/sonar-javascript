Why is this an issue?

If a private member of the class is declared but not used in the program, it can be considered dead code and should therefore be removed. This will improve maintainability because developers will not wonder what the variable is used for.

The private class members were introduced in ES2022 and use # hash symbol prefix. It is possible to declare private fields, methods, getters and setters as well as their static counterparts.

class MyClass {
    #privateField; // Noncompliant, #privateField is unused
    #privateMethod(){} // Noncompliant, #privateMethod is unused
    // ...
}

To fix the code remove unused private member of the class.

class MyClass {
    // ...
}
